Active Directory and Entra ID (Formerly Azure AD) Assessments

Almost all companies that use Microsoft products or services on-premises use Active Directory as their identity provider. Additionally, companies using Microsoft 365 or Azure-based services use Microsoft Entra ID as an identity platform, often in a hybrid configuration.

Since these products are widely used, threat actors frequently exploit common misconfigurations. During an assessment, Outsider Security evaluates the infrastructure’s security posture and provides actionable recommendations.

Active Directory Assessment

Active Directory environments often accumulate insecure settings over years of use. Outsider Security identifies vulnerabilities and misconfigurations frequently abused by attackers, including ransomware operators.

Typical focus areas:

  • Privilege delegation model
  • Admin account protection and separation
  • Least privilege configuration
  • Group policy settings
  • PKI configuration (AD CS)
  • Kerberos configuration

Assessment types:

  • Review: Performed with privileged access to analyze configurations.
  • Penetration Test: Starts from a low-privilege user to escalate access.

Microsoft Entra (Azure AD) Assessment

Entra ID underpins Microsoft 365, Azure resources, and third-party apps. Outsider Security’s Entra ID assessments identify misconfigurations, highlight risks, and recommend improvements.

Example areas investigated:

  • Administrator role usage & MFA adoption
  • Conditional Access configuration
  • Application security
  • Tenant hardening

Additional coverage:

  • Intune for device management
  • Hybrid AD setups
  • Azure RBAC identity usage
